# Local Definition of Done (DoD)

## 🛠 Functional Requirements
- [ ] Logic implemented according to specifications.
- [ ] No "Work-in-Progress" (WIP) code left (no `TODO` comments without tickets).
- [ ] Edge cases (empty states, null-pointers, error handling) are covered.

## 🧪 Quality & Testing
- [ ] Local Dev-Test passed (Happy Path).
- [ ] Unit tests added/updated (focus on business logic, not implementation).
- [ ] No regression: Existing features still work.

## 📂 Documentation & Clean-up
- [ ] `notes.md` is updated with the final solution design.
- [ ] Debug-logs and `console.log` statements removed.
- [ ] Screenshots/Logs attached for proof of work.
- [ ] Folder moved to `/archive` (only after task is moved to 'Done' on the board).
